Overview

Lists filtered records.

Summary

Adds additional operations to Filter API that are specific to Emails. A special macro, $current_user_id, is a convenience that allows for finding emails involving the current user.

Filter Expressions

The additional filters:

Emails Sent By

The $from operation allows you to find emails sent by one or more people. When the person is not known, only the email address is stored. In such a case, the participant_module is EmailAddresses and the participant_id is the ID of the email address from the database. In the example, you would find any emails sent by the current user, the Contacts record with ID "fa300a0e-0ad1-b322-9601-512d0983c19a", or the email address with ID "b0701501-1fab-8ae7-3942-540da93f5017".
Example
{
    "filter":[
        {
            "$from":{
                {
                    "participant_module":"Users",
                    "participant_id":"$current_user_id"
                },
                {
                    "participant_module":"Contacts",
                    "participant_id":"fa300a0e-0ad1-b322-9601-512d0983c19a"
                },
                {
                    "participant_module":"EmailAddresses",
                    "participant_id":"b0701501-1fab-8ae7-3942-540da93f5017"
                }
            }
        }
    ]
}
Below is a list of operation types in addition to those from Filter API:
Operation Description
$from Finds emails sent by one or more people as specified as an array.
