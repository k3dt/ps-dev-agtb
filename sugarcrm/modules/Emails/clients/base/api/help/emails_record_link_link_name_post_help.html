Overview

Creates a record related to an email. See RelateRecordApi for details.

email_addresses_from New email addresses may be created. If the email address already exists, then it will be linked. An email may only have one sender. Creating a new email address replaces the existing sender, if any. The sender cannot be changed after the email is archived.
email_addresses_to One or more email addresses may be created. If the email address already exists, then it will be linked. The recipients cannot be changed after the email is archived.

attachments Create related notes as attachments. filename_guid is used to specify the temporary file ID for an uploaded file that should be attached as a Notes record. upload_id is used to specify the ID of an existing file that the Notes record should reference as the attachment. This allows files to be shared by multiple records. When using upload_id, the file_source should be provided to indicate from where the file originated. The value should be the module of the record that shares its ID with the file. So when an attachment comes from an email template, the file_source should be EmailTemplates. And when an attachment comes from a document, the file_source should be DocumentRevisions.
